How to work with MVC5 and ASP.NET

In completing this project, I created a model (with data to store in a database), and then a controller, made with entity framework, which then created a view. Inside of the the corresponding view, I had an index page, where I put my html for the page, as well as a table to show the data. The data fields are all editable and each field has to have a required input in order to save the data.

The Model

The model is where essentially the database context is created. A database string is then added to the web.config file, where we have a working database for the profile section of the solution.


The Controller

The controller is created by scaffolding, which is automatically run through entity framework. This makes all of the coding simple, because it automatically creates an index, create, edit, and delete method for the data.


The View

The view is what is going to be displayed on the screen. Thus, I went in and put all of my html code inside the index view of the page, which then showed the code whenever you navigated to the profiles tab of the webpage.

The Shared View Layout

Speaking of the profiles tab, I added this tab to the top of the nav-bar through the shared views layout class. Once again, MVC5 and ASP.NET makes this simple to do since it already creates a shared veiw layout class for you to begin with.
